# Bombe Walkthrough

The gameplay in Bombe is based around completing many, _many_, **_MANY_** levels of minesweeper by creating rules around which actions to take based on set conditions.  As we go along, we'll unlock several tools to aid us in defining and abstracting these rules.  The most impactful tool of these is likely the variables, $\{X,\Psi,K,\Pi,H\}$.  In this walkthough, I've broken up the game in a series of "phases" based around working to unlock these variables and applying them to our abstractions.

| Phase | Description |
| ----- | ----------- |
| [Phase 0](phase-0.md) | **Tutorial.** The first set of levels serve as a mostly-silent tutorial prior to unlocking the level set selector. |
| Phase 1 | **Static Rules.** Unlocking the level set selector opens up many options for progression, but we're still limited to static values in our rules. |
| Phase 2 | **1-Variable Rules.** Unlocking the $X$ variable allows us to refine our rules to better match our abstractions. |
| Phase 3 | **2-Variable Rules.** Unlocking the $\Psi$ variable allows us to refine our rules to better match our abstractions. |
| Phase 4 | **3-Variable Rules.** Unlocking the $K$ variable allows us to refine our rules to better match our abstractions. |
| Phase 5 | **4-Variable Rules.** Unlocking the $\Pi$ variable allows us to refine our rules to better match our abstractions. |
| Phase 6 | **5-Variable Rules.** Unlocking the $H$ variable allows us to refine our rules to better match our abstractions. |
